* Overview

This tutorial gets 4G/LTE cellular connectivity working on your OnePlus 6 running NixOS Mobile. It builds on the [[https://github.com/mwlaboratories/phoneputer/tree/main][main branch tutorial]], which covers the initial install — flashing the base image, setting up the phone, and doing your first ~nixos-rebuild~.

*Prerequisites:* You should already have a working NixOS Mobile install on your OnePlus 6 from the main branch tutorial. If not, go complete that first.

The key to getting 4G working is a fork of mobile-nixos at [[https://github.com/mwlaboratories/mobile-nixos/tree/sdm845-bleeding-edge][mwlaboratories/mobile-nixos (sdm845-bleeding-edge)]]. Upstream mobile-nixos ships with a Linux 6.4 kernel and has a broken module system for cross-compiled builds — meaning the modem drivers never actually load. This fork fixes all of that. The fixes were worked out with the help of [[https://claude.ai][Claude]], Anthropic's AI assistant.

* What This Fork Fixes

If you just want 4G working, skip to [[*Rebuild with 4G Support][Rebuild with 4G Support]].

** Kernel Upgrade: 6.4 → 6.19

Upstream mobile-nixos pins the SDM845 kernel to Linux 6.4. This fork upgrades to *Linux 6.19* (linux-next) from [[https://codeberg.org/sdm845/linux][Codeberg's sdm845 tree]], which includes significantly better Qualcomm modem/radio driver support. The kernel config was also reworked — the old static 7,500+ line config file was replaced with dynamic generation from ~defconfig~ + ~sdm845.config~ fragment.

** The Module Fix (This Is Why 4G Didn't Work)

This is the critical fix. Upstream builds the SDM845 kernel with ~isModular = false~, which means kernel modules get *compiled* but *never installed* to the filesystem. The Qualcomm modem stack (QRTR, QMI, RMNET, IPA) depends entirely on kernel modules — so with ~isModular = false~, ~modprobe~ can't find them and the modem never comes up.

This fork sets ~isModular = true~ and fixes several related issues that surfaced:

1. *Module directory version mismatch* — ~modDirVersion~ must exactly match the kernel's release string, or modules install to the wrong directory.
2. *Missing zstd dependency* — Newer kernels compress modules with zstd; without it in build inputs, compression silently fails.
3. *Cross-compilation depmod failure* — When building aarch64 kernels on x86_64, ~make modules_install~ runs ~depmod~ for the host architecture, which silently produces nothing. Without ~modules.dep~, ~modprobe~ refuses to load anything. The fix runs ~depmod~ explicitly in ~postInstall~.

** Other Fixes

- *Sparse image support* — SDM845 bootloaders expect Android sparse format images. The fork adds ~img2simg~ conversion automatically.
- *Firmware handling* — More robust removal of ~modem.mbn~ from the initrd using ~find~ instead of a glob ~rm~ that could fail.
- *Kernel builder updates* — Added ~python3~ to build inputs, relaxed config validations for the newer kernel, and improved ~Image.gz~ handling.

* Rebuild with 4G Support

** Step 1: SSH into Your Phone

#+begin_src sh
ssh root@<your-phone-ip>
#+end_src

Default password from the first tutorial: ~nixtheplanet~

** Step 2: Get the 4G Configuration

If you already have the phoneputer repo cloned on your phone:

#+begin_src sh
cd ~/phoneputer
git fetch origin
git checkout 4g-testing
#+end_src

Or clone fresh:

#+begin_src sh
git clone -b 4g-testing https://github.com/mwlaboratories/phoneputer.git
cd phoneputer
#+end_src

** Step 3: Rebuild and Reboot

The flake on this branch points to the mobile-nixos fork instead of upstream. Rebuild and reboot:

#+begin_src sh
nixos-rebuild switch --flake .#phoneputer
reboot
#+end_src

This will take a while — it's building the new 6.19 kernel with all the module fixes. Insert your SIM card if you haven't already. After reboot, 4G should be available in GNOME Settings → Network.

* What's Different on This Branch

Compared to the main branch, this ~4g-testing~ branch changes only the flake input:

#+begin_src nix
# main branch — upstream mobile-nixos (no 4G)
mobile-nixos.url = "github:mobile-nixos/mobile-nixos";

# 4g-testing branch — fork with kernel + module fixes
mobile-nixos.url = "github:mwlaboratories/mobile-nixos/sdm845-bleeding-edge";
#+end_src

Everything else (~configuration.nix~, ~local.nix~) stays the same. The fixes live entirely in the mobile-nixos fork.
