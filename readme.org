* Overview

This tutorial gets 4G/LTE cellular connectivity working on your OnePlus 6 running NixOS Mobile. It builds on the [[https://github.com/mwlaboratories/phoneputer/tree/main][main branch tutorial]], which covers the initial install — flashing the base image, setting up the phone, and doing your first ~nixos-rebuild~.

*Prerequisites:* You should already have a working NixOS Mobile install on your OnePlus 6 from the main branch tutorial. If not, go complete that first.

The key to getting 4G working is a fork of mobile-nixos at [[https://github.com/mwlaboratories/mobile-nixos/tree/sdm845-bleeding-edge][mwlaboratories/mobile-nixos (sdm845-bleeding-edge)]]. Upstream mobile-nixos ships with a Linux 6.4 kernel and has a broken module system for cross-compiled builds — meaning the modem drivers never actually load. This fork fixes all of that. The fixes were worked out with the help of Claude.

* What This Fork Fixes

** Kernel Upgrade: 6.4 → 6.19

Upstream mobile-nixos pins the SDM845 kernel to Linux 6.4. This fork upgrades to *Linux 6.19* (linux-next) from [[https://codeberg.org/sdm845/linux][Codeberg's sdm845 tree]], which includes significantly better Qualcomm modem/radio driver support. The kernel config was also reworked — the old static 7,500+ line config file was replaced with dynamic generation from ~defconfig~ + ~sdm845.config~ fragment.

** The Module Fix (This Is Why 4G Didn't Work)

This is the critical fix. Upstream builds the SDM845 kernel with ~isModular = false~, which means kernel modules get *compiled* but *never installed* to the filesystem. The Qualcomm modem stack (QRTR, QMI, RMNET, IPA) depends entirely on kernel modules — so with ~isModular = false~, ~modprobe~ can't find them and the modem never comes up.

This fork sets ~isModular = true~ and fixes several related issues that surfaced:

1. *Module directory version mismatch* — ~modDirVersion~ must exactly match the kernel's release string, or modules install to the wrong directory.
2. *Missing zstd dependency* — Newer kernels compress modules with zstd; without it in build inputs, compression silently fails.
3. *Cross-compilation depmod failure* — When building aarch64 kernels on x86_64, ~make modules_install~ runs ~depmod~ for the host architecture, which silently produces nothing. Without ~modules.dep~, ~modprobe~ refuses to load anything. The fix runs ~depmod~ explicitly in ~postInstall~.

** Other Fixes

- *Sparse image support* — SDM845 bootloaders expect Android sparse format images. The fork adds ~img2simg~ conversion automatically.
- *Firmware handling* — More robust removal of ~modem.mbn~ from the initrd using ~find~ instead of a glob ~rm~ that could fail.
- *Kernel builder updates* — Added ~python3~ to build inputs, relaxed config validations for the newer kernel, and improved ~Image.gz~ handling.

* Rebuild with 4G Support

** Step 1: Build the Boot Image (On Your Desktop)

The 6.19 kernel boot image needs to be built on a desktop machine — not on the phone. Your x86_64 workstation needs ~boot.binfmt.emulatedSystems = [ "aarch64-linux" ]~ enabled for cross-building.

Clone the mobile-nixos fork and build the Android fastboot images:

#+begin_src sh
git clone -b sdm845-bleeding-edge https://github.com/mwlaboratories/mobile-nixos.git
cd mobile-nixos
nix-build --argstr device oneplus-enchilada -A outputs.android-fastboot-images
#+end_src

This produces ~result/boot.img~ (kernel + initramfs in Android boot format) and ~result/system.img~ (root filesystem). The build takes a while since it's compiling the full 6.19 kernel.

** Step 2: Put the Phone into Fastboot Mode

*From SSH* (if the phone is currently running NixOS):

#+begin_src sh
ssh root@<your-phone-ip>
reboot bootloader
#+end_src

*Or physically*: power off the phone completely, then hold *Volume Up + Power* simultaneously until you see the "START" screen. Plug in the USB cable after the fastboot screen appears.

** Step 3: Flash Boot Image to Slot B

We flash the new boot image to *slot b only*, keeping slot a untouched as a fallback with the original (non-4G) kernel. Do *not* use ~flash-critical.sh~ here — that script flashes ~--slot=all~ and would overwrite your fallback.

On your desktop, open a shell with Android tools and flash:

#+begin_src sh
nix-shell -p android-tools

# Verify the phone is detected
fastboot devices

# Check which slot is currently active (should be a)
fastboot getvar current-slot

# Flash the new boot image to slot b only
fastboot flash boot_b result/boot.img

# Set slot b as the active boot slot
fastboot set_active b

# Reboot into the new kernel
fastboot reboot
#+end_src

The phone will now boot from slot b with the 6.19 kernel. If anything goes wrong, you can recover by entering fastboot mode (Volume Up + Power) and switching back:

#+begin_src sh
fastboot set_active a
fastboot reboot
#+end_src

** Step 4: Rebuild with the Fork's Kernel Modules

All the 4G fixes live in the mobile-nixos fork — your own ~configuration.nix~ doesn't need to change. The only requirement is that your flake's ~mobile-nixos~ input points to the fork instead of upstream:

#+begin_src nix
mobile-nixos.url = "github:mwlaboratories/mobile-nixos/sdm845-bleeding-edge";
#+end_src

Rebuild remotely from your desktop to speed things up (building the 6.19 kernel on the phone itself is painfully slow):

#+begin_src sh
nixos-rebuild switch --flake .#phoneputer \
  --target-host root@<your-phone-ip> --build-host localhost
#+end_src

This installs the matching kernel modules and regenerates the system configuration. Reboot the phone after it completes.

Insert your SIM card if you haven't already. After reboot, 4G should be available in GNOME Settings → Network.

* What's Different on This Branch

Compared to the main branch, this ~4g-testing~ branch changes only the flake input:

#+begin_src nix
# main branch — upstream mobile-nixos (no 4G)
mobile-nixos.url = "github:mobile-nixos/mobile-nixos";

# 4g-testing branch — fork with kernel + module fixes
mobile-nixos.url = "github:mwlaboratories/mobile-nixos/sdm845-bleeding-edge";
#+end_src

Everything else (~configuration.nix~, ~local.nix~) stays the same. The fixes live entirely in the mobile-nixos fork.
